我曾经想过用编程语言写一篇小说，它的内容类似如下：

```
function wakeup() {
  openEyes()
  turnOffClocks()
  time.sleep(random(10))
}


```

这段代码写的是一个打工人从起床到出门的故事，与其说是故事，更像是流水账，所以小说名字就叫《打工人的一天》。

忘了是什么缘由想写这篇小说。也许是重读卡尔维诺，受到他穷尽小说形式的启发；也许是那段时间工作繁重，日益麻木，以至于某个时刻忽然生出一种恍惚感，活的就像代码运行，早上在闹钟声中开启定时任务，在各个函数里来回跳转，在各种 if else 中按照预定逻辑决策，在抛出的各种异常里处理意料之中的意外，最终一天结束，关机睡觉，第二天继续重复同样的定时任务。

“就像我的生活，从来不会出差错”

不过当时那篇小说没有写完，因为生活中毕竟还是有很多无法提前写好的逻辑，比如同事在饭桌上提到什么话题，我开口说话了，此时我该怎么实现 generateConversion() 呢？

而现在，这个函数很容易实现，调用 ChatGPT 吧。把刚刚聊天的内容输入进去，把我过去对相关话题的见解输入进去，甚至把那天的天气、我的心情统统输进去，我想大模型能够给出一个很符合实际情况的回答。



我相信，只要上下文够长，只要模型能力够强。
